FEATURE OVERVIEW:
─────────────────

When an UNAUTHENTICATED user tries to:
  ✓ Click on a featured property
  ✓ Click "View all" locations
  ✓ Click "Browse Locations"
  ✓ Click "List Your Property"

They will see:
  1. Error toast notification with helpful message
  2. Redirect to sign in page (/auth?tab=signin)
  3. Option to sign up if they don't have account

When an AUTHENTICATED user tries the same actions:
  → They navigate normally without any interruption

CODE IMPLEMENTATION:
────────────────────

Pattern 1: Link with Click Handler (LocationCard)
───────────────────────────────────────────────────
const handleCardClick = (e: React.MouseEvent<HTMLAnchorElement>) => {
  if (!user) {
    e.preventDefault();
    toast.error("Please sign in to view property details");
    navigate("/auth?tab=signin");
    return;
  }
};

<Link to={`/locations/${location.id}`} onClick={handleCardClick}>
  {/* Card content */}
</Link>


Pattern 2: Button with Click Handler (Index)
──────────────────────────────────────────────
<Button
  onClick={() => {
    if (!user) {
      toast.error("Please sign in to browse locations");
      navigate("/auth?tab=signin");
    } else {
      navigate("/locations");
    }
  }}
>
  Browse Locations
</Button>


DEPENDENCIES USED:

✓ useAuth()
  - From: @/contexts/AuthContext
  - Provides: user object (null if not signed in)

✓ useNavigate()

  - Used: Programmatic navigation to auth page

✓ toast.error()
  - From: sonner
  - Used: Display error notifications


USER FLOW:
──────────

UNAUTHENTICATED USER:

  Homepage loaded (user not signed in)
         ↓
  User sees featured properties
         ↓
  User clicks property card
         ↓
  Middleware checks: if (!user)
         ↓
  Toast shown: "Please sign in..."
         ↓
  Redirected to /auth?tab=signin
         ↓
  User signs in/signs up
         ↓
  Can now access all features


AUTHENTICATED USER:

  Homepage loaded (user signed in)
         ↓
  User sees featured properties
         ↓
  User clicks property card
         ↓
  Middleware checks: if (!user) → FALSE (user exists)
         ↓
  Navigation proceeds normally
         ↓
  Property detail page loads
         ↓
  No interruption
